Build Options

Panino requires only one option to be set:

  • path: an array of files and directories for your source

This is either the first argument in the cli version:

panino [path]

Or, the first argument in a script using panino.parse():

panino.parse([path],...)

If you're rendering to HTML, you need one more options:

  • skin: the location of your primary Jade template:

This is added ia --skin in the cli:

panino [path] --skin ./templates/layout.jade

Or, as a build option in panino.render():

panino.parse([path],..., fucntion(err, ast) {
   panino.render(ast, {"skin":"./templates/layout.jade"},...) 
});

However, there are a plentiful number of options to tweak and manage. Provided below are first the cli command, followed by the build option you can use:

  • --exclude | exclude
    Glob patterns of filenames to exclude; (you can use wildcards: ?, , *)
  • -o, --output | output
    Resulting file(s) location.
    Defaults to out
  • -skin | skin
    Location for your primary Jade template; required only for HTML rendering
  • -u, --assets | assets
    A directory containing some asset files for HTML rendering. These are copied to your out directory.
  • --use | use
    Loads a custom plugin for rendering
  • -a, --additionalObjs | additionalObjs
    The path to a JSON file containing a structure defining documentation sources for additional objects. See Linkifying Everything for more information.
  • --alias | aliases
    Registers extensions alias. For example, cc:js registers the cc extension as an alias of js. These are comma-separated.
  • -r, --render | renderer
    Documentation renderer (html, json). More can be added by custom plugins.
    Defaults to HTML
  • --link-format | linkFormat View sources link format. You can use {file} and {line} and any of your package.json variables for interpolation
    Defaults to no links
  • -t, --title | title
    Documentation title template. You can use any of your package.json variables for interpolation.
    Defaults to {package.name} {package.version} API documentation
  • -s, --split | split
    Splits the output into a file per class Defaults to true
  • --package | package
    Read a specified package.json file.
    Defaults to ./package.json
  • -d, --disableTests | disableTests
    Disables the test suite that runs at the end of an HTML build. This is NOT recommended. Defaults to false
  • --index | index
    Index file (with introduction text), e.g. a README.md file.
  • --keepOutDir | keepOutDir
    Does not wipe output directory before building
    Defaults to false
  • --showInternals | showInternals
    Shows members that are marked as internal in the rendered output.
    Defaults to true
